Project Back Story

Note: This is a fictional back story since I don’t know David Allen :-)

You’re uncle’s name is David Allen - yes - that David Allen. You’re in high school and you are complaining to your mom about Uncle David’s Christmas gifts and you tell her “it’s been the same $5 since I was 5 years old followed by the same question ‘what productive thing have you done this year?’”

Your mom asks “well maybe you should read Uncle David’s ‘Getting Things Done’ book and produce something rather than whine about it every year…what do you think?” You think about it, and after mulling it for a while you decide to read through his book (actually, you read through a summary of his book…after all you can only fit so much reading time between Twitter and Snapchat). You’ve also been learning some JavaScript, so you decide to create a simple ToDo application in JavaScript.

Project Details

You can make this project as simple or complex as you want. However, start by making the simplest application possible and then make it more complex if you have time.

Your project will consist of a one page ToDo application written using plain JavaScript, HTML, and CSS (optional). Your JavaScript code can be part of the html page or it can be a separate .js file.

When I say plain JavaScript I mean:

  • use concepts taught in this class
  • use Chapter 8 (advanced battleship) code to help you create the program
  • no jQuery or any similar library
  • no JavaScript framework of any kind (so none of the frameworks listed on todomvc.com or any framework similar to those frameworks)

You are allowed to use your favorite search engine(s) to look for code examples as long as you give credit to the places where you found the examples (see “How to Submit Project” section).

To get an A or Credit you need to meet the following specification:

You will have on input text field and one submit button. When the user enters a todo item (i.e. text into the input text field) and then presses the submit button, then your JavaScript code will put the todo item as part of an unordered list. The list of todo items can be above or below the form.
